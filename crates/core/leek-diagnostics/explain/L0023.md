# L0023: duplicate case

Two `case` labels in the same `switch` test the same value. Because the
first matching label always wins, the second one is unreachable — its body
can never run. This usually means one of the labels was meant to be a
different value.

Erroneous example:

```leekscript
switch (x) {
    case 1:
        return "one"
    case 1:            // unreachable — same label as above
        return "uno"
    default:
        return "?"
}
```

The fix is to give the duplicate label its intended distinct value:

```leekscript
switch (x) {
    case 1:
        return "one"
    case 2:
        return "two"
    default:
        return "?"
}
```

Only side-effect-free labels (literals, names, arithmetic) are compared.
A label that calls a function is left alone, since two calls might
legitimately produce different values. The `default` arm is never
considered a duplicate.
