# E0253: duplicated map key

A map literal repeats the same key. Later entries silently overwrite
earlier ones, so the first value is dead — almost always a
copy-paste slip. Checked at LeekScript 4.

Erroneous example:

```leekscript
var damages = [
    "fire": 40,
    "ice": 25,
    "fire": 60,         // duplicate map key `fire`
]
```

Remove or rename the duplicate:

```leekscript
var damages = [
    "fire": 60,
    "ice": 25,
]
```

Only keys that are literally identical are flagged; two expressions
that merely evaluate to the same value at runtime are not. The
analogous check for `switch` statements is the L0023 lint
(duplicate case).
