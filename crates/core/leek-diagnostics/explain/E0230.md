# E0230: break or continue outside a loop

A `break` appears outside any loop or `switch`, or a `continue`
appears outside any loop. These statements redirect a loop's control
flow; with no enclosing loop there is nowhere to jump.

Erroneous example:

```leekscript
function check(x) {
    if (x < 0) {
        continue        // no loop to continue
    }
    return x
}
```

Use the control flow that exists at this level — usually `return`:

```leekscript
function check(x) {
    if (x < 0) {
        return 0
    }
    return x
}
```

Note that a *lambda* body is its own function: a `break` inside a
lambda passed to `arrayMap` does not belong to the loop that the
mapping conceptually performs. `break` is legal inside `switch` arms
(it ends the switch); `continue` is not.
