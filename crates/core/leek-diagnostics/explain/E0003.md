# E0003: unexpected character

The lexer hit a character that cannot start any LeekScript token —
for example `#`, `§`, or a stray Unicode symbol pasted from a chat or
a document. The character is skipped and reported; everything after
it lexes normally.

Erroneous example:

```leekscript
var x = 5 § 3
```

Remove or replace the character:

```leekscript
var x = 5 % 3
```

A frequent source is invisible characters from copy-paste — smart
quotes (`“”`) instead of `"`, non-breaking spaces, or zero-width
characters. The diagnostic prints the offending character in quoted
debug form so look-alikes are distinguishable. Re-typing the
surrounding code by hand usually clears it.
