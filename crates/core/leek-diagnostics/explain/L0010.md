# L0010: self comparison

Both sides of a comparison are the same side-effect-free expression:
`x == x`, `a < a`, `this.n != this.n`. Such a comparison is constant —
always true for `==`, `<=`, `>=`, always false for `!=`, `<`, `>` —
so it never tests anything. It is usually a typo for a different
variable.

Erroneous example:

```leekscript
if (myCell == myCell) {   // always true; did you mean enemyCell?
    moveToward(enemy)
}
```

Compare against the value you meant:

```leekscript
if (myCell == enemyCell) {
    moveToward(enemy)
}
```

Expressions with side effects (function calls, `++`) are exempt, since
evaluating them twice could legitimately differ. The note on the
diagnostic tells you whether the comparison is always true or always
false. Silence an intentional use with `// @allow(L0010)`.
