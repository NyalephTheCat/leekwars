# L0014: identical operands

A binary operator was applied to two structurally identical operands. For
most operators the result is a constant or a tautology, which signals a
typo — usually one side should have referred to a different variable.

Erroneous example:

```leekscript
if (x == x) { … }       // always true
var d = width - width   // always 0
var m = flags & flags   // just flags
```

The fix is to correct the operand that was meant to differ:

```leekscript
if (x == y) { … }
var d = width - height
```

For the *bitwise* operators `&` and `|`, `a & a` and `a | a` are exactly
`a` (idempotent), so the linter offers a machine-applicable fix that
replaces the expression with the single operand. The *logical* operators
`&&` and `||` are **not** simplified automatically: in Leekscript they
coerce their operands, so `x && x` is not always equal to `x` (for
example `0 && 0` is `false`, not `0`). Those cases are flagged for you to
review by hand.
