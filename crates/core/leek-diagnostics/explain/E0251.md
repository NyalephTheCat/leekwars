# E0251: incompatible type

A function's `return` statement doesn't match the function's
declared return type. This fires in three shapes:

- `return value` where the value's type isn't assignable to the
  declared return type;
- a bare `return` in a function declared to return something;
- `return value` in a function declared `void`.

Erroneous example:

```leekscript
integer half(x) {
    return x / 2.0      // function returns integer, but value is real
}
```

Make the value and the declaration agree:

```leekscript
real half(x) {
    return x / 2.0
}
```

A function with no declared return type accepts any `return`. As
with assignments, `integer` and `real` are mutually convertible —
the example above is flagged only when the checker is certain the
value is `real`.
