# L0013: double negation

An expression is negated twice: `!!x` or `not not x`. Double negation
booleanizes the value but reads as noise — in a condition, the value
is already evaluated for truthiness, so the two `!`s cancel out.

Erroneous example:

```leekscript
if (!!visible) {
    say("spotted")
}
```

Drop both negations:

```leekscript
if (visible) {
    say("spotted")
}
```

The lint ships a machine-applicable autofix that strips the leading
`!!`. A single negation is of course never flagged, and `!(!a || b)`
style expressions where the inner `!` belongs to a larger operand are
not double negations. Silence with `// @allow(L0013)` if you use `!!`
deliberately as a to-boolean conversion.
