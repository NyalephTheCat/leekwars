# E0266: instruction after a terminator

A statement appears after a `return`, `break`, or `continue` in the
same block. Control flow has already left the block, so the
statement can never execute. Upstream LeekScript treats this as an
error, not a warning.

Erroneous example:

```leekscript
function damage(target) {
    return getLife(target)
    debug("unreachable")    // cannot add instruction after a terminator
}
```

Move the statement before the terminator, or delete it:

```leekscript
function damage(target) {
    debug("checking target")
    return getLife(target)
}
```

Only one error is reported per block, even if several dead
statements follow. This error covers the *same-block* case only;
unreachable code created by other shapes (a condition that is always
false, a loop that never runs) is reported by the L0005 lint
instead.
