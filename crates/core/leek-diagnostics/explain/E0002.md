# E0002: invalid number

A numeric literal is malformed. The most common shape is a non-digit
suffix glued onto the number — `123abc`, `0b102`, `0xfg` — where the
lexer can tell the trailing characters were meant to be part of the
literal but aren't valid digits for its base.

Erroneous example:

```leekscript
var a = 123abc      // letters glued onto a decimal literal
var b = 0b102       // `2` is not a binary digit
```

Separate the identifier from the number, or fix the digits:

```leekscript
var a = 123
var b = 0b101
```

Binary literals (`0b…`) accept only `0` and `1`; hex literals (`0x…`)
accept `0-9a-f`. An underscore separator is allowed between digits
for readability (`1_000_000`) — but see E0004 for the rules on
consecutive separators.
