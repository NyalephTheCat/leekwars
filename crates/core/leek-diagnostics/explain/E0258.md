# E0258: duplicate enum variant

An `enum` declaration lists the same variant name twice. Each
variant must be unique — a repeated name would shadow the earlier
one and its value would be unreachable.

Erroneous example:

```leekscript
enum Color {
    RED,
    GREEN,
    RED,                // duplicate variant `RED` in enum `Color`
}
```

Remove or rename the duplicate:

```leekscript
enum Color {
    RED,
    GREEN,
    BLUE = 10,
}
```

Variants are integer-backed (auto-numbered from 0 unless given an
explicit `= value`) and are accessed as `Color.RED`. Enums are an
experimental feature (enabled with the `LEEK_EXPERIMENTAL_ENUMS`
environment variable).
