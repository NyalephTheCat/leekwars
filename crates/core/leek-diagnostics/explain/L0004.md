# L0004: empty block

An `if`, `else`, `while`, `for`, `foreach`, or `do-while` has an empty
`{}` body. The construct evaluates its condition and then does
nothing — most often a leftover stub from an edit, or a branch whose
body was deleted but whose header survived.

Erroneous example:

```leekscript
if (enemy == null) {
}
while (getTP() > 0) {}   // spins the condition, does nothing
```

Either fill the body in or remove the construct:

```leekscript
if (enemy == null) {
    return
}
```

This is a hint, not a warning: an empty body is occasionally
intentional (for example, a loop whose condition does all the work).
For those cases, a comment inside the block does not silence the lint —
use `// @allow(L0004)` on the preceding line instead.
