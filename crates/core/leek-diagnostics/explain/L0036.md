# L0036: useless foreach write

A `foreach` binding is assigned inside the loop body, but the binding
is by-value: each iteration gets a fresh *copy* of the element, so
the write modifies the copy and the collection is left unchanged.

Erroneous example:

```leekscript
for (var x in arr) {
    x = x * 2        // modifies the copy — arr is unchanged
}
```

LeekScript's `@` reference bindings are the language's own answer:

```leekscript
for (var @x in arr) {
    x = x * 2        // x aliases the element — this rewrites arr
}
```

If the write was never meant to escape the iteration (the variable is
used as a scratch value), rename it to a new local instead so the
intent is clear. Reads of the binding are fine; only assignments to a
by-value binding are flagged, and `@` bindings are never flagged.
This is a *nursery* lint: it only runs with `miku lint --nursery` or
`nursery = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0036)`.
