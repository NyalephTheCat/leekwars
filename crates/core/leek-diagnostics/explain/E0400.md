# E0400: manifest parse error

The project's `Miku.toml` manifest could not be parsed, or a field
has the wrong shape. The message carries the underlying TOML or
validation error.

Example:

```text
miku: Miku.toml: project.version must be a string
```

A minimal valid manifest looks like:

```toml
[project]
name = "my-ai"
entry = "src/main.leek"
version = "4"
```

Common causes:

- `version` written as a bare number (`version = 4`) instead of a
  string (`version = "4"`);
- TOML syntax errors — unclosed quotes, missing `]` on a section
  header, duplicate keys;
- a misspelled section or key name.

Tooling falls back to standalone-file mode when no manifest is found
(e.g. `miku fmt path/to/file.leek`), but a manifest that *exists*
must parse — this error is not skippable.
