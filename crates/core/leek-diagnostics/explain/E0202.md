# E0202: redeclared symbol

A name is declared twice in the same scope — two `var`s, two
functions, or two classes with the same name. The second declaration
doesn't create a new variable; it collides with the first, and which
one later code means is ambiguous.

Erroneous example:

```leekscript
var target = getNearestEnemy()
var target = getNearestAlly()    // already declared above
```

Either reuse the existing binding or pick a new name:

```leekscript
var target = getNearestEnemy()
target = getNearestAlly()        // plain reassignment
```

Re-*assigning* is always fine; only re-*declaring* is an error. The
same code fires for duplicate function declarations (unless the
experimental overloads feature is enabled) and duplicate class names.
Declaring the same name in a *nested* scope is a different situation —
see E0212 (shadowing) and the L0003 lint.
