# E0502: invalid edit range

A text edit produced by a rewrite pass has an inverted range — its
start position is greater than its end position — so it cannot
describe a region of the source.

Example message:

```text
error[E0502]: invalid edit range 42..17
```

Like the other edit errors, this is an internal consistency check of
the rewriting machinery, not a problem with your LeekScript code. A
malformed range indicates a bug in whichever pass produced the edit
(a formatter rewrite, a lint auto-fix, or a migration pass) — report
it with the input file that triggers it.

Related: E0500 (overlapping edits), E0501 (edit past end of file).
