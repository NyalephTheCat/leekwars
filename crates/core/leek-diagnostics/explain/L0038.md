# L0038: shadowed builtin

A variable, parameter, or function declaration reuses the name of a
builtin function. From that point on the local wins, so a later call
like `count(items)` stops meaning what it says — a confusing failure
mode, especially for readers who know the builtin catalog.

Erroneous example:

```leekscript
var count = 0              // shadows the builtin count(...)
function debug(x) {        // shadows the builtin debug(...)
    // ...
}
```

Pick a name that doesn't collide:

```leekscript
var enemyCount = 0
function logState(x) {
    // ...
}
```

This is the standard-library sibling of L0003 (shadowed binding):
that lint compares against outer locals, this one against builtins.
This is a *nursery* lint: it only runs with `miku lint --nursery` or
`nursery = true` under `[lint]` in Miku.toml. Silence a deliberate
override with `// @allow(L0038)`.
