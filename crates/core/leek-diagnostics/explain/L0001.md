# L0001: unused variable

A local variable was declared and possibly assigned, but its value is
never read before it goes out of scope. The binding does nothing, and is
usually either a leftover from an edit or a sign that a later use was
forgotten.

Erroneous example:

```leekscript
function distance(a, b) {
    var dx = b - a
    var dy = 0        // declared, never read
    return abs(dx)
}
```

The fix is to remove the binding:

```leekscript
function distance(a, b) {
    var dx = b - a
    return abs(dx)
}
```

If the value is computed for a side effect you want to keep, drop the
`var` and keep the expression statement. If the variable is intentionally
unused (for example, a destructuring placeholder), prefix it with an
underscore — `var _unused = …` — which silences this lint.

Suppress a single occurrence with `// @allow(L0001)` (or
`// @allow(unused-variable)`) on the line above the declaration.
