# L0019: redundant ternary

A conditional expression that does nothing a plainer expression
wouldn't do. Two shapes are flagged:

- `cond ? a : a` — both arms are identical, so the result is `a` no
  matter what `cond` evaluates to.
- `cond ? true : false` (or `cond ? false : true`) — this just yields
  the boolean value of `cond` (or its negation).

Erroneous example:

```leekscript
var cell = ready ? target : target     // always `target`
var ok = (hp > 50) ? true : false      // just `hp > 50`
```

Use the value directly:

```leekscript
var cell = target
var ok = hp > 50
```

The identical-arms form is auto-fixed to the arm value when the
condition is side-effect-free (otherwise the condition's effects would
be lost). The boolean-literal form is flagged without a fix, since the
correct rewrite depends on whether `cond` is already a boolean.
Silence with `// @allow(L0019)`.
