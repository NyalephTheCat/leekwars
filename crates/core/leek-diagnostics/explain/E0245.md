# E0245: duplicated method

A class defines the same method name twice with overlapping arity —
both definitions could match the same call, so the second one is
ambiguous.

Erroneous example:

```leekscript
class Brain {
    act(target) {
        useWeapon(target)
    }

    act(target) {           // method `act` is already defined
        useChip(getChips()[0], target)
    }
}
```

Overloading *by arity* is allowed — keep the signatures disjoint or
merge the bodies:

```leekscript
class Brain {
    act(target) {
        useWeapon(target)
    }

    act(target, chip) {      // distinct arity: fine
        useChip(chip, target)
    }
}
```

The overlap check accounts for default parameters: `act(a, b = 1)`
accepts both 1 and 2 arguments, so it overlaps with both `act(a)`
and `act(a, b)`. The same rule applies to constructors (E0265).
