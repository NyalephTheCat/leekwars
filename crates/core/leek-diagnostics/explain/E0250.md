# E0250: incompatible assignment type

A value is assigned to a variable whose declared type cannot hold
it. Typed declarations (LeekScript 4) fix the variable's type for
its whole lifetime; later assignments must stay compatible.

Erroneous example:

```leekscript
integer n = 5
n = "five"              // cannot assign string value to `n` of type integer
```

Keep the types consistent, or declare the variable with the type you
actually need:

```leekscript
integer n = 5
n = 6

var anything = 5        // untyped: can hold any value
anything = "five"
```

Related strict-mode cases reported under the same code: applying a
compound assignment (`a *= 5`) or an index assignment (`a[0] = 1`)
to a variable currently bound to `null`, and inserting into a
collection that was declared empty. `integer` and `real` values
convert freely between each other; everything else must match the
declared type.
