# E0240: private field

A field declared `private` is accessed from outside the class that
owns it. Private members are only visible inside the declaring
class's own methods and constructors.

Erroneous example:

```leekscript
class Tank {
    private armor = 50
}

var t = new Tank()
debug(t.armor)          // field `armor` is private on class `Tank`
```

Expose the value through a public method (or make the field public):

```leekscript
class Tank {
    private armor = 50

    getArmor() {
        return this.armor
    }
}

var t = new Tank()
debug(t.getArmor())
```

`private` is stricter than `protected`: even subclasses cannot read a
parent's private field (that case is reported with the owning
ancestor's name). If subclass access is intended, use `protected`
instead — see E0243.
