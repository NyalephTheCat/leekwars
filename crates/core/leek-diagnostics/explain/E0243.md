# E0243: protected field

A field declared `protected` is accessed from outside the class
hierarchy that owns it. Protected members are visible to the
declaring class *and its subclasses*, but not to unrelated code.

Erroneous example:

```leekscript
class Unit {
    protected energy = 10
}

var u = new Unit()
debug(u.energy)         // field `energy` is protected on class `Unit`
```

Access it from within the hierarchy, or through a public accessor:

```leekscript
class Unit {
    protected energy = 10
}

class Bulb extends Unit {
    report() {
        debug(this.energy)   // fine: subclass access
    }
}

new Bulb().report()
```

If even subclasses should not touch the field, use `private` (E0240).
If everything may, declare it `public` or leave the modifier off.
