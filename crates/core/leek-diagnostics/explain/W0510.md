# W0510: migration compile break

`miku migrate` rewrote the source for the target LeekScript
version, but the result no longer compiles there — some construct in
the file has no automatic rewrite, and after migration it trips a
real error at the new version. The warning wraps that error:

```text
warning[W0510]: migrated source no longer compiles at @version:4: [E0261] ...
  note: this construct has no automatic rewrite for the target
        version — migrate it by hand
```

This only fires when the *original* file compiled cleanly at its old
version — the migration genuinely broke something, rather than
carrying over a pre-existing error.

What to do: open the migrated file at the reported location and port
the construct manually; the embedded error code (in brackets) tells
you exactly what the target version objects to — `miku explain` that
code for details.

The companion warnings are W0511 (compiles, but behaves differently)
and W0512 (a construct the migrator refuses to touch).
