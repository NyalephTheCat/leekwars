# L0024: unnecessary else

An `else` follows an `if` whose branch always exits the enclosing block —
it ends in `return`, `break`, or `continue`. The `else` adds a level of
nesting that buys nothing: control only reaches the code after the `if`
when the condition was false, so the `else` body can be dedented to follow
the `if` directly.

Erroneous example:

```leekscript
function classify(x) {
    if (x < 0) {
        return "negative"
    } else {                 // unnecessary
        return "non-negative"
    }
}
```

The fix is to drop the `else` and pull its body up one level:

```leekscript
function classify(x) {
    if (x < 0) {
        return "negative"
    }
    return "non-negative"
}
```

This is a readability hint. It does not apply to `else if` chains, which
are guard sequences rather than redundant nesting, and it does not fire
when the `if` branch can fall through (does not always exit).
