# L0035: string concatenation in loop

A string is built by repeated concatenation inside a loop. Each `+=`
copies everything accumulated so far, so the loop costs O(n²)
operations as the string grows — a hidden ops-budget sink in debug
and logging code.

Erroneous example:

```leekscript
var msg = ""
for (var e in enemies) {
    msg += getName(e) + ", "   // copies the whole string every pass
}
```

Collect the pieces and join once — that's linear:

```leekscript
var parts = []
for (var e in enemies) {
    push(parts, getName(e))
}
var msg = join(parts, ", ")
```

The detection heuristic looks for a compound `+=` (or `x = x + …`)
inside a loop whose right side contains a string literal — the
literal is the tell that this is string building, since the HIR is
untyped and purely-variable concats can't be classified. This is a
*nursery* lint: it only runs with `miku lint --nursery` or
`nursery = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0035)`.
