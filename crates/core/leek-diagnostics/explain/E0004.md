# E0004: multiple numeric separators

A numeric literal uses two or more `_` separators in a row.
Underscores are allowed *between* digits to group long numbers, but
consecutive underscores (`1__000`) are rejected: they are usually a
typo, and silently accepting them would make visually different
literals compare equal.

Erroneous example:

```leekscript
var budget = 1__000_000
```

Use single separators:

```leekscript
var budget = 1_000_000
```

Separators carry no value — `1_000` and `1000` are the same number.
They are valid in decimal, binary, and hex literals as long as each
underscore sits between two digits with no neighbor underscore.
