# L0009: duplicate branches

The `then` and `else` arms of an `if` are structurally identical. The
condition makes no difference to what happens, which is almost always
a copy-paste bug — one arm was meant to be edited and wasn't.

Erroneous example:

```leekscript
var chip = getChips()[0]
if (getLife() < 50) {
    useChip(chip, getEntity())
} else {
    useChip(chip, getEntity())        // same as the then-branch
}
```

Either make the arms differ, or drop the conditional:

```leekscript
useChip(getChips()[0], getEntity())
```

The comparison ignores spans and is binding-aware: arms that declare
their own local variables are compared by structure, not by name, so
two arms that merely look similar but bind different things are not
reported. Silence a deliberate duplicate with `// @allow(L0009)`.
