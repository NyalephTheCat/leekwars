# L0020: duplicate include

A file `include`s the same script more than once. Includes are
idempotent — the first `include` pulls the file in and later ones are
no-ops — so the duplicate is dead weight, usually left behind after
moving code around.

Erroneous example:

```leekscript
include("utils")
include("combat")
include("utils")    // already included above
```

Remove the duplicate:

```leekscript
include("utils")
include("combat")
```

The lint ships an autofix that removes the second `include`. Only
exact duplicates within the same file are flagged — including a file
that another included file also includes is handled by the include
system itself and is not reported here. Silence with
`// @allow(L0020)`.
