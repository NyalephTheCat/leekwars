# L0041: map used as a set

A map only ever stores `true` as its values: it's a set wearing a map
costume. Every write is `m[k] = true`, every read asks whether a key
is present — the values never matter. LeekScript 4 sets state that
directly and skip storing the dummy values.

Erroneous example:

```leekscript
var seen = [:]
for (var cell in cells) {
    seen[cell] = true        // value never matters
}
if (target in seen) { ... }
```

Use a set:

```leekscript
var seen = <>
for (var cell in cells) {
    setPut(seen, cell)
}
if (target in seen) { ... }
```

Detection is per-body: a local declared as an empty map literal (or
one whose values are all `true`) where *every* use is set-shaped —
`m[k] = true` writes, `m[k]` / `k in m` / `mapContainsKey` reads,
`mapRemove`, and size queries. A single non-conforming use (a
non-`true` value, the map passed to another call, stored elsewhere)
disqualifies the candidate entirely. Gated on LeekScript version 4+.
This is a *nursery* lint: it only runs with `miku lint --nursery` or
`nursery = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0041)`.
