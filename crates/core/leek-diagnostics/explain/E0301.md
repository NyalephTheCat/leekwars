# E0301: backend directive not allowed

A `@<backend>-backend:` directive appears in a doc comment on a
function where directives aren't honored. Backend directives are
only read from *bodiless function signatures* in signature files
(the experimental function-signatures mode); anywhere else they are
dropped with this warning so they can't silently change how normal
code compiles.

Erroneous example:

```leekscript
// @native-backend: intrinsic
function dist(a, b) {           // has a body: directive ignored
    return getCellDistance(a, b)
}
```

Either remove the directive from the regular function, or move it
onto a bodiless signature in a signature file (with the
`LEEK_EXPERIMENTAL_FN_SIGNATURES` feature enabled):

```leekscript
// @native-backend: intrinsic
function dist(a, b) -> integer;
```

The warning is informational — compilation proceeds with the
directive ignored.
