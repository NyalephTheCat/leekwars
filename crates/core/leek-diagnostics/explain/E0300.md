# E0300: lowering unsupported

The compiler's lowering stage (AST → HIR → MIR) hit a construct it
cannot translate. Unlike most errors, this usually points at a
compiler limitation or at code that earlier stages let through in a
shape the lowerer can't model — for example an assignment target
that isn't a recognizable l-value, or a reference to a local the
lowerer never saw bound.

Example messages:

```text
error[E0300]: assignment target is not an l-value MIR knows how to model
error[E0300]: lambda captures unbound local x; passing null instead
```

What to do:

- Check the surrounding code for constructs that are unusual but
  technically parse — restructuring the expression (splitting a
  complex assignment, binding an intermediate to a variable) usually
  sidesteps the limitation.
- If the code looks reasonable and other tools (`miku check`)
  accept it, the lowering gap is worth reporting as a bug.

This error comes from the compilation pipeline (`miku build` /
`miku run`), not from static analysis — `miku check` alone may not
surface it.
