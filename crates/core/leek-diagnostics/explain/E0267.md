# E0267: unknown method

A `super.method(...)` call names a method that no ancestor class
declares. `super` calls resolve statically against the inheritance
chain, so the target method must exist on some parent at
compile time.

Erroneous example:

```leekscript
class Unit {
    move() {
        moveToward(getNearestEnemy())
    }
}

class Scout extends Unit {
    move() {
        super.advance()     // unknown method `advance` on class `Unit`
        say("scouting")
    }
}
```

Call the method the parent actually has:

```leekscript
class Scout extends Unit {
    move() {
        super.move()
        say("scouting")
    }
}
```

If an ancestor *does* declare the method but with a different number
of parameters, the error is E0220 (invalid parameter count) instead.
Calls on ordinary objects (`obj.method(...)`) are checked separately
and report E0242 when the member doesn't exist.
