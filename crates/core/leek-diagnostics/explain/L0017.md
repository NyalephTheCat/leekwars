# L0017: duplicate condition

An `if … else if` chain tests the same side-effect-free condition
twice. Arms are tried in order, so a later arm whose condition is
identical to an earlier one can never run — the earlier branch always
wins. This usually means one of the conditions was meant to differ.

Erroneous example:

```leekscript
if (dist < 3) {
    useWeapon(target)
} else if (dist < 10) {
    useChip(chip, target)
} else if (dist < 3) {   // already handled by the first arm
    moveAwayFrom(target)
}
```

Fix the condition that was meant to be different:

```leekscript
if (dist < 3) {
    useWeapon(target)
} else if (dist < 10) {
    useChip(chip, target)
} else if (dist >= 30) {
    moveAwayFrom(target)
}
```

Conditions with side effects (function calls that may return different
values on each evaluation) are not compared, so no false positives
arise from `if (roll()) … else if (roll())`. Silence with
`// @allow(L0017)`.
