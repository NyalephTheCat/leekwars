# L0006: constant condition

The condition of an `if`, `while`, or ternary always evaluates to the same
value, so the branch is either always taken or always dead. This is almost
always a mistake — a comparison against the wrong operand, or a leftover
debugging value.

Erroneous example:

```leekscript
if (true) {        // condition is always true
    doThing()
}

while (1 == 1) {   // never terminates
    step()
}
```

The fix depends on intent. If you meant a real test, supply it:

```leekscript
if (ready) {
    doThing()
}
```

A deliberately infinite loop is clearer written as `while (true)` with an
explicit `break`, which this lint allows — the warning targets conditions
that are *accidentally* constant (comparisons of two literals, a variable
compared to itself, etc.), not intentional `while (true)` loops.
