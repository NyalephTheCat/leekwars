# L0040: array literal membership

A membership test runs against an array literal: `cell in [1, 5, 9]`
or `inArray([1, 5, 9], cell)`. An array is an *ordered sequence*;
using one for "is this value one of these?" both reads wrong and
scans linearly.

Erroneous example:

```leekscript
if (cell in [1, 5, 9]) {
    moveTowardCell(cell)
}
```

LeekScript 4 set literals are the structure built for membership —
hash lookup, and the intent is in the type:

```leekscript
if (cell in <1, 5, 9>) {
    moveTowardCell(cell)
}
```

Membership against an array *variable* is not flagged — the variable
may legitimately be an ordered sequence used in many ways; only
literal arrays whose sole purpose is the test are reported. Gated on
LeekScript version 4+ — older scripts don't have sets. This is a
*nursery* lint: it only runs with `miku lint --nursery` or
`nursery = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0040)`.
