# L0005: unreachable code

A statement appears after a `return`, `break`, or `continue` in the
same block. Control flow has already left the block by the time the
statement would run, so it can never execute. This usually means a
statement was left behind during a refactor, or an early exit was
inserted above code that still matters.

Erroneous example:

```leekscript
function damage(x) {
    return x * 2
    debug("computed")   // never runs
}
```

Move the live code above the exit, or delete it:

```leekscript
function damage(x) {
    debug("computing")
    return x * 2
}
```

Only one finding is reported per unreachable region — the first dead
statement is flagged, not every statement after it. Conditional exits
(`if (a) { return }`) don't count as definite terminators, so code
after them is not flagged even when every path returns. Silence an
intentional case with `// @allow(L0005)`.
