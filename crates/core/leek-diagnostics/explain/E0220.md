# E0220: invalid parameter count

A function, method, or `super` call passes a number of arguments the
callee cannot accept. The diagnostic names the expected arity, taking
default parameters into account — a function with defaults accepts a
*range* of argument counts.

Erroneous example:

```leekscript
function dist(a, b) { return abs(a - b) }
dist(1, 2, 3)            // expects 2 args; got 3
```

Match the signature:

```leekscript
dist(1, 2)
```

Parameters with default values may be omitted from the right:
`function f(a, b = 2)` accepts both `f(1)` and `f(1, 5)`. Builtins
are checked the same way against their catalog arity. If you control
the function and the extra information is real, add a parameter (or a
default) rather than passing it positionally and hoping.
