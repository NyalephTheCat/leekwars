# E0231: duplicated parameter

A function declares the same parameter name twice. Both parameters
would bind the same name, so one of the passed arguments would be
unreachable.

Erroneous example:

```leekscript
function hit(target, target) {   // duplicate parameter `target`
    useWeapon(target)
}
```

Give each parameter a distinct name:

```leekscript
function hit(target, weapon) {
    setWeapon(weapon)
    useWeapon(target)
}
```

This applies to functions, methods, constructors, and lambdas alike.
A parameter may still shadow an outer variable of the same name —
that's a separate concern (see E0212 and the L0003 lint).
