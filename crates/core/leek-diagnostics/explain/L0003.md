# L0003: shadowed binding

A local variable or parameter declares a name that is already bound by
an enclosing scope. The inner binding *shadows* the outer one: until
the inner scope ends, the name refers to the new variable, and any
write to it is lost when the scope closes. This is usually an
accidental re-declaration rather than a deliberate choice.

Erroneous example:

```leekscript
var i = 0
for (var i = 0; i < 10; i++) {  // inner `i` shadows the outer one
    // ...
}
// `i` is still 0 here
```

Rename one of the bindings (or reuse the outer one) so each name means
one thing:

```leekscript
var total = 0
for (var i = 0; i < 10; i++) {
    total += i
}
```

Shadowing a `global` is not flagged — globals live in their own
namespace and re-declaring the name locally is usually intentional.
Lambda parameters are also exempt, since callback signatures often
force a name. Silence an intentional case with `// @allow(L0003)` on
the line above the declaration.
