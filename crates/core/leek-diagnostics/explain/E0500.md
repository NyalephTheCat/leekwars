# E0500: edit overlap

Two text edits produced by a rewrite pass cover overlapping byte
ranges of the same source. Overlapping edits have no well-defined
application order, so the whole edit set is rejected rather than
applied ambiguously.

Example message:

```text
error[E0500]: edit 10..25 overlaps existing edit 18..30
```

This is an internal consistency error of the rewriting machinery
(`miku fmt` rewrites, lint auto-fixes, `miku migrate` passes) — it
means two fixes targeted the same code. What to do:

- Apply fixes one at a time (re-run the tool after each apply); the
  second fix will compute against the already-rewritten source.
- If a single tool emits overlapping edits on its own in one run,
  that's a bug worth reporting with the input that triggers it.

The related errors are E0501 (an edit that points past the end of
the file) and E0502 (an edit whose range is inverted).
