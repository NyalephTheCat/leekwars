# E0222: cannot assign to final field

A field declared `final` is assigned outside of the class's
constructor. A final field gets its value once — at its declaration
or inside a constructor — and is read-only afterwards.

Erroneous example:

```leekscript
class Config {
    final limit = 12
}

var c = new Config()
c.limit = 20            // cannot assign to final field
```

Drop the `final` modifier if the field needs to change, or set the
value in the constructor:

```leekscript
class Config {
    final limit

    constructor(limit) {
        this.limit = limit
    }
}

var c = new Config(20)
```

The same error fires for the builtin final fields such as `Real.PI`,
`Integer.MAX_VALUE`, or `String.EMPTY` — those can never be
reassigned. For bare builtin constants like `PI` or `INFINITY`, see
E0210 instead.
