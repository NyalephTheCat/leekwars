# E0260: default argument not at end

A parameter without a default value follows a parameter that has one.
Arguments are matched positionally, so once one parameter is
optional, everything after it must be optional too — otherwise a call
like `f(1)` is ambiguous about which parameter the `1` fills.

Erroneous example:

```leekscript
function fire(power = 10, target) {   // `target` has no default
    useChip(power, target)
}
```

Put the defaulted parameters last:

```leekscript
function fire(target, power = 10) {
    useChip(power, target)
}
```

Any number of defaulted parameters is fine as long as they form the
tail of the list: `function f(a, b = 1, c = 2)` accepts `f(x)`,
`f(x, y)`, and `f(x, y, z)`.
