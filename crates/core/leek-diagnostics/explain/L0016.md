# L0016: division by zero

A division or modulo has a divisor that is statically known to be zero.
At runtime this yields no useful value and almost certainly indicates a
mistake — a wrong constant, or a variable that should have been used in
place of the literal.

Erroneous example:

```leekscript
var ratio = total / 0
var rem = n % (3 - 3)
```

The fix is to use the intended non-zero divisor:

```leekscript
var ratio = total / count
```

If the divisor can legitimately be zero at runtime, guard it before the
division so the zero case is handled explicitly:

```leekscript
if (count != 0) {
    ratio = total / count
}
```

This lint only fires when the divisor folds to a literal zero. A divisor
that is merely *possibly* zero (an unconstrained variable) is not reported
here — add your own guard where the value's range is unknown.
