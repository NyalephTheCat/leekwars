# L0032: needless index loop

A counting `for` loop's counter is only ever used to index a single
array. A `foreach` loop says what the iteration is *about* (the
elements), cannot go out of bounds, and skips the per-iteration
`count()` call and index arithmetic.

Erroneous example:

```leekscript
for (var i = 0; i < count(enemies); i++) {
    useWeapon(enemies[i])   // `i` exists only to read enemies[i]
}
```

Iterate the elements directly:

```leekscript
for (var enemy in enemies) {
    useWeapon(enemy)
}
```

The rule is deliberately conservative: the counter must start at 0
and step by `++` or `+= 1`, every use of it must read `arr[i]` for
one fixed side-effect-free array, and any *write* through `arr[i]`
bails out (a by-value `foreach` binding would not update the array).
This is a *pedantic* lint: it only runs with `miku lint --pedantic` or
`pedantic = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0032)`.
