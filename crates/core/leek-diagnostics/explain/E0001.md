# E0001: string not closed

A string literal is still open when the end of the file is reached.
The lexer scanned forward looking for the closing quote and never
found one — usually because a quote was deleted, or because the
string's own content contains an unescaped quote that ended the
string earlier than intended, leaving a later quote to open a new,
unterminated one.

Erroneous example:

```leekscript
var msg = "hello
```

Close the string:

```leekscript
var msg = "hello"
```

Both `"double"` and `'single'` quoted strings are affected. When the
error's span starts somewhere unexpected, look just *before* it: an
unescaped quote earlier in the file may have shifted every subsequent
string boundary. Write a literal quote inside a string by escaping it
(`"he said \"hi\""`) or by using the other quote style
(`'he said "hi"'`).
