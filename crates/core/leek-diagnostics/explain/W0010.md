# W0010: unknown pragma

A line comment looks like a pragma (`// @something`) but the name
isn't one the compiler recognizes. The line is ignored, and this
warning points out that it had no effect — useful when a real pragma
was misspelled.

Erroneous example:

```leekscript
// @verison:4        <- typo: had no effect
var x = 1
```

Fix the spelling:

```leekscript
// @version:4
var x = 1
```

Recognized pragmas are `@version:N` (language version 1–4), `@strict`
(strict mode), and `@experimental:feature` (opt in to an experimental
feature; this warning also fires when `@experimental` is missing its
feature name). Ordinary comments that happen to start with `@` — like
doc-comment tags — are only scanned at the start of a line comment,
so `/** @param x */` blocks are unaffected.
