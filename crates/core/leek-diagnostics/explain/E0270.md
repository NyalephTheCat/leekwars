# E0270: AI or library not existing

An `import` statement could not be resolved. This error has three
forms:

1. **Imports not enabled.** `import` is an experimental feature; the
   file must opt in with a pragma:

   ```leekscript
   import fight.generator      // `import` is experimental; enable
                               // `// @experimental:imports`
   ```

   Fix by adding the pragma at the top of the file:

   ```leekscript
   // @experimental:imports
   import fight.generator
   ```

2. **Invalid import statement.** The statement parsed but doesn't
   name a well-formed library path.

3. **Unknown library.** The path is well-formed but no builtin
   library has that name:

   ```leekscript
   // @experimental:imports
   import fight.simulator      // builtin library `fight.simulator`
                               // does not exist
   ```

A successful import injects the library's symbols into scope — for
example `import fight.generator` (alias: `fight_generator`) provides
`fightGenerate`, `fightSeed`, and `fightPreview`. Note that `import`
is unrelated to `include("file")`, which splices another source file
of your project and reports E0272/E0273/E0274 on failure.
