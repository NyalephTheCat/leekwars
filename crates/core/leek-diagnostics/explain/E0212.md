# E0212: variable name unavailable

A declaration uses a name that is reserved or already taken by an
enclosing scope. Two situations produce this error:

- a `var` declaration shadows a binding from an outer scope —
  upstream LeekScript makes this a hard error, not a style choice;
- a class is declared with a reserved name (built-in type names like
  `Number`, `String`, `Array`, `Value`, …).

Erroneous example:

```leekscript
var cell = getCell()
function move() {
    var cell = getCell()     // shadows the outer `cell`
}
```

Pick a distinct name:

```leekscript
var cell = getCell()
function move() {
    var targetCell = getCell()
}
```

Shadowing a *builtin* name is allowed (user code may define its own
`search`), so only collisions with outer user bindings and reserved
class names are rejected.
