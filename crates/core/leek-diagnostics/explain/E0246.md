# E0246: private constructor

`new` is used on a class whose matching constructor is declared
`private`. A private constructor can only be invoked from inside the
class itself — the usual pattern for singletons or static factory
methods.

Erroneous example:

```leekscript
class Game {
    private constructor() {}
}

var g = new Game()      // constructor of class `Game` is private
```

Go through the class's own factory, or widen the constructor:

```leekscript
class Game {
    static instance = null

    private constructor() {}

    static get() {
        if (Game.instance == null) {
            Game.instance = new Game()
        }
        return Game.instance
    }
}

var g = Game.get()
```

A `protected` constructor is the softer variant — subclasses may
still construct via `super(...)` — and is reported as E0262 when
violated.
