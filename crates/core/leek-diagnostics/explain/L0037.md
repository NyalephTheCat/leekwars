# L0037: interval loop

A classic three-clause counting loop can be written as a LeekScript 4
interval iteration. The interval form has no header to get subtly
wrong — no `<=` vs `<` confusion, no `i++` vs `++j` typos — and reads
as "every i in this range".

Erroneous example:

```leekscript
for (var i = 0; i <= 10; i++) { debug(i) }
for (var j = 0; j < n; j++) { debug(j) }
```

Use intervals (`[a..b]` includes the bound, `[a..b[` excludes it):

```leekscript
for (var i in [0..10]) { debug(i) }
for (var j in [0..n[) { debug(j) }
```

Only plain count-ups are flagged: the start must be a literal, the
bound `<` or `<=`, the step `++`, and the body must never write the
counter (an interval can't be fast-forwarded mid-flight). The lint is
gated on LeekScript version 4+ — older scripts don't have intervals.
This is a *nursery* lint: it only runs with `miku lint --nursery` or
`nursery = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0037)`.
