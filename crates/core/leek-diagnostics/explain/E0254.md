# E0254: impossible cast

A function is passed as an argument where the receiving parameter is
declared `Function<... => R>`, but the passed function's return type
cannot be `R`. The call could never produce a value of the expected
type, so the cast is impossible.

Erroneous example:

```leekscript
string label() {
    return "leek"
}

function apply(Function< => integer> f) {
    return f()
}

apply(label)    // cannot pass function `label` returning string
                // where integer return is expected
```

Pass a function with a compatible return type:

```leekscript
integer score() {
    return 42
}

apply(score)
```

`integer` and `real` returns are mutually acceptable. Note that a
function with *no* return value cannot satisfy a parameter expecting
a `Function<... => integer>` — "returns nothing" is not a valid
integer-returning function, even though `null` is assignable in most
other positions.
