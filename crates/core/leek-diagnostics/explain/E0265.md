# E0265: duplicated constructor

A class declares two constructors whose arities overlap — a call
could match both, so the second declaration is ambiguous.

Erroneous example:

```leekscript
class Vec {
    constructor(x) {
        this.x = x
    }

    constructor(x) {        // constructor already defined for 1 argument
        this.x = x * 2
    }

    x = 0
}
```

Constructors may be overloaded *by arity* — keep the signatures
disjoint:

```leekscript
class Vec {
    constructor(x) {
        this.x = x
    }

    constructor(x, scale) {     // distinct arity: fine
        this.x = x * scale
    }

    x = 0
}
```

Default parameters widen a constructor's accepted range:
`constructor(x, y = 0)` accepts 1 and 2 arguments and therefore
overlaps with both `constructor(x)` and `constructor(x, y)`. The
same overlap rule applies to methods (E0245).
