# L0039: manual range check

A pair of comparisons tests whether a value lies in a range:
`0 <= x && x < 10`, or the negated form `x < 0 || x > 10`. LeekScript
4 interval membership says the same thing in one operator, and the
bracket carries the inclusivity — no `<` vs `<=` typo surface.

Erroneous example:

```leekscript
if (0 <= x && x < 10) { moveToward(enemy) }
if (x < 0 || x > 10) { moveAwayFrom(enemy) }
```

Use interval membership (`[a..b]` inclusive, `[a..b[` half-open):

```leekscript
if (x in [0..10[) { moveToward(enemy) }
if (x not in [0..10]) { moveAwayFrom(enemy) }
```

Detection requires both comparisons to bound the *same*
side-effect-free expression from below and above. Side effects
disqualify the match because `&&`/`||` short-circuit the second
comparison while an interval evaluates both bounds. Gated on
LeekScript version 4+. This is a *nursery* lint: it only runs with
`miku lint --nursery` or `nursery = true` under `[lint]` in Miku.toml.
Silence with `// @allow(L0039)`.
