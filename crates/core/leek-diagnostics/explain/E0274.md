# E0274: included file failed to parse

An `include("name")` statement resolved to a readable file, but that
file contains syntax errors and could not be parsed. The error is
attached at the `include(...)` call site; the included file's own
diagnostics pinpoint the actual syntax problem.

Example:

```leekscript
// main.leek
include("utils")        // included file `utils` failed to parse
```

```leekscript
// utils.leek
function helper( {      // syntax error here
    return 1
}
```

Fix the syntax errors in the included file — run `miku check` (or
open the file in an editor with the LSP) to see them directly.

Each included file is parsed with its *own* `// @version:N` pragma,
so a file written for an older LeekScript version keeps its original
syntax rules even when included from a newer file. See E0272 and
E0273 for the not-found and unreadable variants.
