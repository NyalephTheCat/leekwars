# E0262: protected constructor

`new` is used on a class whose matching constructor is declared
`protected`. A protected constructor can be invoked by the class
itself and by its subclasses (through `super(...)` or `new` inside
their own methods), but not by unrelated code.

Erroneous example:

```leekscript
class Shape {
    protected constructor() {}
}

var s = new Shape()     // constructor of class `Shape` is protected
```

Instantiate a concrete subclass instead — the subclass constructor
may chain to the protected parent constructor:

```leekscript
class Shape {
    protected constructor() {}
}

class Circle extends Shape {
    constructor() {
        super()
    }
}

var s = new Circle()
```

This is the usual way to make a class "abstract": unrelated code
cannot create a bare `Shape`, but subclasses build on it freely. The
stricter variant is a `private` constructor (E0246), which even
subclasses cannot call.
