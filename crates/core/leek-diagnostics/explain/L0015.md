# L0015: assignment in condition

An assignment (`=`) is used where a condition is expected: `if (x = 5)`,
`while (n = next())`. This is almost always a typo for `==` — the
assignment stores the value and then tests the *stored value's*
truthiness, so `if (x = 5)` is always true and quietly overwrites `x`.

Erroneous example:

```leekscript
if (state = 2) {      // assigns 2 to state, always true
    useWeapon(getNearestEnemy())
}
```

Use a comparison — or, if the assignment is intentional, hoist it out
of the condition so the intent is visible:

```leekscript
if (state == 2) {
    useWeapon(getNearestEnemy())
}
```

The few legitimate uses (assign-and-test loops) read more clearly with
the assignment as its own statement, so this lint always warns rather
than trying to guess intent. Silence a deliberate case with
`// @allow(L0015)`.
