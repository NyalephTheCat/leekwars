# L0021: negated comparison

A comparison was wrapped in a logical-not (`!(a == b)`). This reads more
clearly as the directly negated comparison, and avoids the extra
parentheses and the easily-missed leading `!`.

Erroneous example:

```leekscript
if (!(a == b)) { … }
if (!(x < y)) { … }
```

The fix is to push the negation into the operator:

```leekscript
if (a != b) { … }
if (x >= y) { … }
```

The rewrites are:

| written            | clearer as |
|--------------------|------------|
| `!(a == b)`        | `a != b`   |
| `!(a != b)`        | `a == b`   |
| `!(a < b)`         | `a >= b`   |
| `!(a <= b)`        | `a > b`    |
| `!(a > b)`         | `a <= b`   |
| `!(a >= b)`        | `a < b`    |

This is a readability hint, not a correctness issue — the two forms are
equivalent. It does not apply to `!` on non-comparison expressions such
as `!flag`, which are already idiomatic.
