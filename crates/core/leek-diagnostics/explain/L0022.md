# L0022: unused expression

An expression is used as a statement, but its result is neither bound,
returned, nor passed anywhere, and evaluating it has no side effect. The
computed value is silently discarded. The most common cause is writing
`==` (comparison) where `=` (assignment) was meant.

Erroneous example:

```leekscript
function f(x) {
    x == 5        // did you mean `x = 5`?
    a + b         // result thrown away
    return x
}
```

The fix depends on intent:

```leekscript
function f(x) {
    x = 5             // assignment
    var sum = a + b   // bind the result
    return x
}
```

A statement that *does* something — a function call, `new`, `++`/`--`, or
any assignment — is never flagged.

Note that Leekscript implicitly returns the **trailing** expression of a
block (`var x = 5\nx + 1` evaluates to `6`). The last statement of every
block is therefore exempt: its value may be the block's result. Only
non-terminal expression statements are reported.
