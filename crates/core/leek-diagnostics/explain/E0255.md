# E0255: type alias not capitalized

A `type` alias declaration uses a name that starts with a lowercase
letter. Alias names follow the class-name convention: only a
capitalized name is recognized in type positions, so a lowercase
alias would be silently ignored — every use would resolve to `any`.
This warning makes that failure loud.

Erroneous example:

```leekscript
type coord = integer    // type alias `coord` is ignored — alias
                        // names must start with an uppercase letter
coord x = 1
```

Capitalize the alias:

```leekscript
type Coord = integer
Coord x = 1
```

`type` aliases are an experimental feature (enabled with the
`LEEK_EXPERIMENTAL_TYPES` environment variable); without the feature
flag, `type` declarations are not parsed at all. Aliases may name
unions too: `type Id = integer | string`.
