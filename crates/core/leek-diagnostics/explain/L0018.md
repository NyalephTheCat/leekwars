# L0018: unused parameter

A function, method, or constructor parameter is never read in its
body. The caller computes and passes a value that the function then
ignores — often a sign the parameter is obsolete, or that a use was
forgotten.

Erroneous example:

```leekscript
function damage(base, shield) {   // `shield` is never read
    return base * 2
}
```

Remove the parameter (and update call sites), or use it:

```leekscript
function damage(base, shield) {
    return max(0, base * 2 - shield)
}
```

This is a hint, not a warning: signatures sometimes must keep a
parameter to match a callback shape or an overridden method. Prefix
the name with an underscore — `_weapon` — to mark it intentionally
unused and silence the lint. Lambda parameters are never flagged,
since their arity is dictated by the caller (`arrayMap` passes the
index whether you want it or not).
