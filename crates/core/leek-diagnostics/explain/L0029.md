# L0029: switch missing default

A `switch` has no `default` arm. When none of the `case`s match,
execution silently falls through the whole statement — which may be
exactly what you want, but nothing in the code says so.

Erroneous example:

```leekscript
switch (state) {
    case 1: useWeapon(target); break
    case 2: useChip(chip, target); break
}
```

Add an explicit `default`, even if it only documents the fall-through:

```leekscript
switch (state) {
    case 1: useWeapon(target); break
    case 2: useChip(chip, target); break
    default: debug("unhandled state: " + state)
}
```

An empty `default:` with a comment is enough — the point is that a
reader (and the linter) can tell the unmatched case was considered.
This is a *pedantic* lint: it only runs with `miku lint --pedantic` or
`pedantic = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0029)`.
