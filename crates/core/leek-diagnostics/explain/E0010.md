# E0010: duplicate pragma

A `@version` or `@strict` pragma appears more than once in the same
file. These pragmas configure how the whole file is lexed and
checked, so a second occurrence is ambiguous — which one wins? The
first occurrence stays in effect and the duplicate is rejected.

Erroneous example:

```leekscript
// @version:3
// @version:4
var x = 1
```

Keep a single directive:

```leekscript
// @version:4
var x = 1
```

Pragmas are line comments of the shape `// @name` or `// @name:value`
and are resolved before lexing (the active version decides which
words are keywords). `@experimental` is exempt from this rule — it
may appear multiple times, each adding a feature to the set.
