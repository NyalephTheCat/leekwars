# L0012: redundant boolean

A value is compared against a boolean literal: `x == true`,
`x != false`, `flag == false`. The comparison adds nothing — it
evaluates to the operand itself (or its negation) and just makes the
condition longer.

Erroneous example:

```leekscript
if (isAlive(enemy) == true) {
    useWeapon(enemy)
}
if (ready != true) {
    moveAwayFrom(enemy)
}
```

Use the operand directly, negating where needed:

```leekscript
if (isAlive(enemy)) {
    useWeapon(enemy)
}
if (!ready) {
    moveAwayFrom(enemy)
}
```

`x == true` and `x != false` simplify to `x`; `x == false` and
`x != true` simplify to `!x`. The lint ships a machine-applicable
autofix for both directions. Silence with `// @allow(L0012)` if you
prefer the explicit comparison.
