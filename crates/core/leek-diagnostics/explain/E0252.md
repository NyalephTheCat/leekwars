# E0252: wrong argument type

An argument passed to a builtin function has a type the builtin
cannot accept. This check runs in strict mode and covers a small,
deliberately conservative set of builtins whose signatures are
known precisely.

Erroneous example:

```leekscript
// @version:4
var n = count(12)       // `count` argument #1: expected array, got integer
```

Pass a value of the expected type:

```leekscript
// @version:4
var n = count([1, 2, 3])
```

Signatures are version-aware: at v1-v3, `count` also accepts maps
and sets; at v4 it is array-only (use `mapSize` / `setSize` for the
others). Booleans count as numeric — LeekScript coerces them to 0/1
— so `abs(true)` is accepted. Values whose type the checker cannot
pin down are never flagged.
