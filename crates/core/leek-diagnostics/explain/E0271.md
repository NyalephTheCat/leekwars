# E0271: circular include

Following the `include("...")` statements from the entry file leads
back to a file that is already being included — the include graph
has a cycle, so there is no valid order in which to load the files.

Erroneous example:

```leekscript
// a.leek
include("b")
function helperA() { return 1 }
```

```leekscript
// b.leek
include("a")            // circular include involving `a`
function helperB() { return helperA() + 1 }
```

Break the cycle by extracting the shared pieces into a third file
both can include:

```leekscript
// common.leek
function helperA() { return 1 }
```

```leekscript
// b.leek
include("common")
function helperB() { return helperA() + 1 }
```

Note that including the same file twice along *different* paths is
fine — each file is loaded once and the second include is a no-op
(the L0020 lint flags literal duplicate includes in a single file).
Only a true cycle is an error.
