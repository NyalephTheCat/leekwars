# E0211: cannot redefine function

A user-declared function's name is being reassigned or
incremented/decremented. Function declarations are not variables:
`function f() { … }` binds `f` permanently, and `f = …` or `f++`
would silently destroy the function.

Erroneous example:

```leekscript
function attack() { useWeapon(getNearestEnemy()) }
attack = null          // cannot reassign a function
```

If you need a rebindable function value, store a lambda in a
variable instead:

```leekscript
var attack = function() { useWeapon(getNearestEnemy()) }
attack = function() { useChip(getChips()[0], getNearestEnemy()) }
```

Calling the function, passing it as a value (`arrayMap(a, attack)`),
and referencing it are all fine — only assignment to the declared
name is rejected.
