# E0501: edit out of bounds

A text edit produced by a rewrite pass ends past the end of the
source it is being applied to.

Example message:

```text
error[E0501]: edit end 512 exceeds source length 300
```

This almost always means the edit was computed against a *different
version* of the file than the one it's being applied to — the file
changed on disk between analysis and application, or an edit set was
replayed onto already-rewritten text.

What to do: re-run the tool so the edits are recomputed against the
current file contents. If the error reproduces in a single run of
one tool, report it as a bug.

Related: E0500 (overlapping edits), E0502 (inverted edit range).
