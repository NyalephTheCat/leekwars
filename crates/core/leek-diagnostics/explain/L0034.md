# L0034: count in loop condition

A size builtin like `count(arr)` is called inside a loop condition.
The condition re-runs on **every** iteration, so the builtin's
operation cost is paid n times even though the size never changes.
Hoisting the call into a variable before the loop pays it once — an
easy win against the fight's ops budget.

Erroneous example:

```leekscript
for (var i = 0; i < count(chips); i++) {
    useChip(chips[i], target)
}
```

Hoist the count:

```leekscript
var n = count(chips)
for (var i = 0; i < n; i++) {
    useChip(chips[i], target)
}
```

The rule is conservative: if the loop body passes the counted
collection to *any* call, the body might grow or shrink it, making
the per-iteration re-count load-bearing — nothing is flagged in that
case. This is a *nursery* lint: it only runs with
`miku lint --nursery` or `nursery = true` under `[lint]` in Miku.toml.
Silence with `// @allow(L0034)`.
