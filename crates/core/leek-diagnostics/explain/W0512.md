# W0512: migration unsupported

`miku migrate` found a construct that simply does not exist at the
target LeekScript version and cannot be rewritten automatically. The
construct is left in place and flagged.

Example — downgrading to v1, which has no classes:

```text
warning[W0512]: v1 has no classes — this declaration cannot be
                downgraded automatically
  note: restructure the class as plain functions over arrays before
        targeting v1
```

`new` expressions get the same treatment (v1 evaluates `new X()` to
`null`).

What to do: restructure the flagged code by hand using the
facilities the target version *does* have — the note on each warning
suggests the usual replacement. Until then, the migrated file will
not work correctly at the target version.

See also W0510 (migrated source no longer compiles) and W0511
(compiles but behaves differently).
