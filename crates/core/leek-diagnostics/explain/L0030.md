# L0030: manual min/max

A ternary re-implements the `min` or `max` builtin by hand:
`a > b ? a : b` is `max(a, b)`; `a < b ? a : b` is `min(a, b)`. The
builtin states what is *meant* instead of how it's computed, and
evaluates each operand exactly once.

Erroneous example:

```leekscript
var dmg = base > cap ? cap : base
```

Use the builtin:

```leekscript
var dmg = min(base, cap)
```

Only side-effect-free operands are matched — when an operand is a
function call, the condition and the arm could legitimately observe
different values, so nothing is flagged. All four orderings (`<`,
`<=`, `>`, `>=`, arms in either order) are recognized. This is a
*pedantic* lint: it only runs with `miku lint --pedantic` or
`pedantic = true` under `[lint]` in Miku.toml. Silence with
`// @allow(L0030)`.
