# E0100: unexpected token

The parser encountered a token that doesn't fit the grammar at this
position. This is the general syntax error — its causes range from a
missing closing bracket to a keyword used where an expression was
expected.

Erroneous example:

```leekscript
var x = (1 + 2
var y = if
```

Balance the delimiters and complete the expression:

```leekscript
var x = (1 + 2)
var y = x > 0 ? x : 0
```

When the reported location looks fine, the real mistake is usually
*earlier*: an unclosed `(`, `[`, or `{` makes the parser misread
everything that follows, and the first token that can't fit gets the
blame. Also check the file's `@version` pragma — syntax such as
classes (v3+) or intervals and sets (v4) parses only in the versions
that have it, so valid-looking code can be "unexpected" to an older
grammar.
